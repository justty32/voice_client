# GEMINI.md: AI 助手開發實驗室與架構設計書

## 專案目標：混合型智能助手 (Hybrid-Agent-Assist)
本專案採「實驗導向開發」模式，將實驗成功的技術整合進最終的 AI 助手架構。

---

## 🏗️ 核心架構藍圖 (演進中)

### 1. 安全邊界與沙盒規範 (Sandbox & Security) —— **最高優先級**
- **目錄隔離 (Chroot-like)：** 助手的讀取、寫入及執行權限僅限於特定的工作目錄 (Workspace Root)。禁止存取任何父目錄或系統敏感路徑。
- **網路隔離：** 助手執行的程式碼或任務流不具備網路存取權限（除了與本地模型及雲端 API 必要的安全連線外，不允許任意存取網際網路）。
- **權限最小化：** 助手執行的指令僅限於白名單，且無法存取系統層級的資源（如硬體、其他應用程式資料等）。

### 2. 核心驅動：事件驅動與 Session 管理 (The Core)
- **Session 系統：** 管理 String Array。每當陣列中有新字串加入，即觸發助手處理鏈條。
- **被動觸發：** 助手不會主動執行，而是根據 Session 列表的新增項目進行反應。

### 3. 原子能力層 (Atomic Capabilities)
- **檔案操作 (CRUD)：** 僅限於沙盒目錄內的文字檔操作。
- **受限 Shell 指令：** `ls`, `touch`, `mkdir`, `mv`, `cp`, `rm`, `grep`, `cat` (嚴禁其他危險指令)。

### 4. 流量調度與執行層 (Hybrid Execution)
- **本地執行 (Ollama 10B)：** 初步解析、意圖路由、執行簡單指令。
- **雲端提權 (Gemini API)：** 處理複雜推理任務。

## 📐 開發規範
1. **路徑檢查：** 每次執行檔案操作前，必須檢查該路徑是否位於沙盒目錄內。
2. **無狀態執行：** 每個處理任務應盡量保持無狀態 (Stateless)，僅依賴當前 Session 內容。
3. **審核日誌：** 紀錄助手執行的所有 Shell 指令與路徑變動，以便後續稽核。
