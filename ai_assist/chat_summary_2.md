# 本地 10B 模型 AI Agent 開發策略總結

這份摘要針對在本地環境使用 Ollama 10B 規模（如 Llama 3 8B, Gemma 2 9B 等）「較笨」的模型開發 AI Agent 提供核心架構與實踐建議。

---

## 一、 核心思路：從「大腦」轉向「零件」
10B 規模模型的長處在於執行明確指令與格式化數據，而非複雜推理。
*   **定位：** 擔任「流程協調員」或「特定任務執行者」。
*   **策略：** 限制模型的自由度，將其視為嚴格執行標準作業程序 (SOP) 的自動化零件。

---

## 二、 推薦的開發框架與工具

### 1. 強化輸出穩定性 (防範格式崩潰)
*   **DSPy:** 透過「編譯」提示語簽名，自動優化 Few-shot 範例，讓小模型表現逼近大模型。
*   **Outlines / Guidance:** 直接干預 Token 生成過程，強制模型輸出 100% 符合 JSON Schema 或正則表達式的格式。
*   **PydanticAI:** 利用 Pydantic 定義結構化輸出，確保 Agent 調用工具時參數正確。

### 2. 架構管理與流程控制
*   **LangGraph:** 建立明確的「狀態機」流程圖。當模型出錯時，可強制其回到上一步重來，避免錯誤累積。
*   **Intent Routing (意圖路由):** 先讓模型做簡單的多選一判斷，再進入縮減過 System Prompt 的子任務分支。

---

## 三、 架構設計四原則
1.  **模組化任務鏈 (Sequential Chain):** 將大任務拆解成多個微小步驟，每步只讓模型做一件小事。
2.  **強制 JSON 輸出:** 使用 Ollama 的 `format: json` 模式配合後端解析。
3.  **多模型協作 (Multi-Agent Swarm):** 採用「執行者 (Worker) + 審核者 (Reviewer)」模式，利用笨模型「擅長挑錯」的特性提升穩定性。
4.  **反思機制 (Self-Reflection):** 若輸出不符預期，自動將結果丟回給模型進行修正。

---

## 四、 實踐技巧與避坑指南
*   **簡短 Prompt:** 將 System Prompt 控制在 500 字以內，減少模型負擔。
*   **上下文管理:** 10B 模型處理超過 4k Token 時注意力會潰散，必須搭配 RAG (向量資料庫) 處理長文本。
*   **高效 Embedding:** 不要用 10B 模型做向量化，改用專門的小模型（如 `bge-m3` 或 `nomic-embed-text`），速度更快且更精確。
*   **量化選擇:** 建議使用 `Q4_K_M` 或 `Q8_0` 量化版本，以平衡推理速度與智能水準。

---

## 五、 適合本地 Agent 的應用場景
*   **隱私級 RAG 助手：** 處理個人文件、代碼庫與筆記。
*   **特定自動化工作流：** 郵件分類、網頁剪報整理、本地文件歸檔。
*   **系統控制中心：** 將自然語言轉化為磁碟清理、播放音樂等具體 API 指令。
*   **數據預處理：** 作為大型雲端系統的前置過濾器，進行實體提取或意圖判斷。
